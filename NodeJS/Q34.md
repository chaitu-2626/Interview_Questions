## ‚úÖ Main Difference Between Browser and Node.js Global Scope

| Feature                          | Browser                            | Node.js                                |
| -------------------------------- | ---------------------------------- | -------------------------------------- |
| **Global Object**                | `window` or `self`                 | `global`                               |
| **Top-level `this`**             | `window` (i.e., `this === window`) | `{}` (not equal to `global`)           |
| **Global variables auto-attach** | Yes (`var x = 1` ‚Üí `window.x`)     | No (`var x = 1` stays local to module) |
| **Scope Isolation**              | All scripts share one global scope | Each file has its **own module scope** |
| **Common Environment**           | Browser (DOM, events, UI)          | Server-side (filesystem, process, etc) |

---

## üß™ Example Comparison:

### üîπ In the **Browser**:

```html
<script>
  var name = 'Alice';
  console.log(window.name); // Alice ‚úÖ
  console.log(this === window); // true ‚úÖ
</script>
```

### üîπ In **Node.js**:

```js
var name = 'Bob';

console.log(global.name); // undefined ‚ùå
console.log(this === global); // false ‚ùå
```

Even though `name` looks like a global variable, it‚Äôs actually **scoped to the current module** in Node.js.

---

## üß† Why This Difference?

* **Browser**: Designed to have all scripts share one global scope (`window`).
* **Node.js**: Each file is treated as a **module**, with its own local scope to avoid variable conflicts.

---

## üß† Interview One-Liner:

> ‚ÄúIn the browser, global variables attach to `window`, and all scripts share one global scope. In Node.js, each file has its own module scope, and the global object is `global`, not `this`. Variables declared at the top of a Node.js file are not truly global unless explicitly set on `global`.‚Äù
