# üßæ JavaScript Hoisting Cheat Sheet

## üîç What is Hoisting?

> **Hoisting** is JavaScript‚Äôs default behavior of **moving declarations** (variables and functions) to the **top of their scope** before code execution.

* **Only declarations are hoisted, not initializations!**
* Happens **during the compile phase** before execution.

---

## üìã What gets Hoisted?

| Declaration Type      | Hoisted?                                            | Initialized?                 | Scope                    |
| --------------------- | --------------------------------------------------- | ---------------------------- | ------------------------ |
| `var`                 | ‚úÖ Yes                                               | ‚úÖ Initialized as `undefined` | Function/Global          |
| `let`                 | ‚úÖ Yes                                               | ‚ùå Not initialized (TDZ)      | Block                    |
| `const`               | ‚úÖ Yes                                               | ‚ùå Not initialized (TDZ)      | Block                    |
| Function Declarations | ‚úÖ Yes                                               | ‚úÖ Fully initialized          | Function/Global          |
| Function Expressions  | ‚ùå No (if assigned to var, var hoisted as undefined) | N/A                          | Depends on var/let/const |

---

## üß† Hoisting Examples

### 1. `var` Hoisting

```js
console.log(x); // undefined (not ReferenceError!)
var x = 5;

var y;
console.log(y); // undefined
y = 10;
```

*Declaration* hoisted; *initialization* stays in place.

---

### 2. `let` and `const` Hoisting (TDZ)

```js
console.log(a); // ReferenceError: Cannot access 'a' before initialization
let a = 3;

console.log(b); // ReferenceError
const b = 4;
```

Variables are hoisted but in the **Temporal Dead Zone (TDZ)** until declared.

---

### 3. Function Declaration Hoisting

```js
sayHi();

function sayHi() {
  console.log("Hello!");
}
// ‚úÖ Works because function declaration is fully hoisted
```

---

### 4. Function Expression Hoisting

```js
sayHello(); // TypeError: sayHello is not a function

var sayHello = function() {
  console.log("Hi!");
};
```

* `sayHello` variable hoisted as `undefined`
* Function expression **not** hoisted

---

## üö© Temporal Dead Zone (TDZ)

* Time between entering scope and declaration of `let`/`const` variable
* Accessing in TDZ throws ReferenceError

---

## üîÑ Hoisting Summary

```js
// Actual execution order behind the scenes:
var x;                 // Declaration hoisted, init with undefined
function foo() {}      // Fully hoisted

console.log(x);        // undefined
foo();                 // works fine
x = 10;                // assignment stays here
```
