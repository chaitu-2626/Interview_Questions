## üîç What is Memoization?

> **Memoization** is an optimization technique that **caches the results** of expensive function calls so that **subsequent calls with the same arguments** return the cached result instantly.

---

## üß† Why Use It?

‚úÖ Avoid redundant calculations
‚úÖ Improve performance (especially for recursion)
‚úÖ Useful for **pure functions** (same input ‚Üí same output)

---

## üì¶ Basic Example

```js
function memoize(fn) {
  const cache = {};
  return function (...args) {
    const key = JSON.stringify(args);
    if (cache[key]) {
      return cache[key]; // return cached result
    }
    const result = fn(...args);
    cache[key] = result;
    return result;
  };
}

// Usage
const slowSquare = n => {
  console.log('Calculating...');
  return n * n;
};

const memoizedSquare = memoize(slowSquare);

memoizedSquare(5); // Calculates
memoizedSquare(5); // Cached
```

## üß™ Real-World Example: Fibonacci

### ‚ùå Without Memoization (inefficient)

```js
function fib(n) {
  if (n <= 1) return n;
  return fib(n - 1) + fib(n - 2);
}
```

‚õî Exponential time complexity

### ‚úÖ With Memoization

```js
function memoizedFib(n, cache = {}) {
  if (n in cache) return cache[n];
  if (n <= 1) return n;
  return (cache[n] = memoizedFib(n - 1, cache) + memoizedFib(n - 2, cache));
}
```

‚ö° Much faster for large inputs

## üìö Memoization vs Caching

| Concept     | Description                            |
| ----------- | -------------------------------------- |
| Memoization | Caching **function results**           |
| Caching     | General technique for **storing data** |
