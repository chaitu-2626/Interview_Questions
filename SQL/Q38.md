## ‚úÖ MySQL Triggers: Concept + Practical Example

### üìå What is a Trigger?

A **trigger** in MySQL is a **set of instructions that automatically runs** when a specific database event (like `INSERT`, `UPDATE`, or `DELETE`) happens on a table.

---

### üîÑ When Triggers Run

| Trigger Type | Description                                     |
| ------------ | ----------------------------------------------- |
| `BEFORE`     | Runs **before** the event (e.g., before insert) |
| `AFTER`      | Runs **after** the event (e.g., after update)   |

| Trigger Event | What it watches for  |
| ------------- | -------------------- |
| `INSERT`      | Row is being added   |
| `UPDATE`      | Row is being changed |
| `DELETE`      | Row is being removed |

## üë∑ Practical Example: Audit Salary Changes in `employee` Table

### üß± Step 1: Create the Tables

```sql
CREATE TABLE employee (
  id INT PRIMARY KEY AUTO_INCREMENT,
  name VARCHAR(100),
  salary DECIMAL(10,2),
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
);

CREATE TABLE employee_audit (
  audit_id INT PRIMARY KEY AUTO_INCREMENT,
  employee_id INT,
  old_salary DECIMAL(10,2),
  new_salary DECIMAL(10,2),
  changed_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

---

### üß† Step 2: Create the Trigger

```sql
DELIMITER //

CREATE TRIGGER after_salary_update
AFTER UPDATE ON employee
FOR EACH ROW
BEGIN
  IF OLD.salary <> NEW.salary THEN
    INSERT INTO employee_audit(employee_id, old_salary, new_salary)
    VALUES (OLD.id, OLD.salary, NEW.salary);
  END IF;
END;
//

DELIMITER ;
```

---

### üß™ Step 3: Test It in Node.js (or SQL)

#### Update employee salary in Node.js:

```js
const mysql = require('mysql2/promise');

async function updateSalary(employeeId, newSalary) {
  const connection = await mysql.createConnection({
    host: 'localhost',
    user: 'your_user',
    password: 'your_password',
    database: 'your_database'
  });

  await connection.execute(
    'UPDATE employee SET salary = ? WHERE id = ?',
    [newSalary, employeeId]
  );

  const [auditLog] = await connection.execute(
    'SELECT * FROM employee_audit WHERE employee_id = ?',
    [employeeId]
  );

  console.log('Audit Log:', auditLog);
  await connection.end();
}
```

> üìå This logs the old and new salary **without writing audit logic in your app** ‚Äì the trigger handles it automatically.

## ‚úÖ Benefits of Using Triggers

* ‚úÖ Automates auditing (e.g., track changes)
* ‚úÖ Enforces business rules (e.g., validation)
* ‚úÖ Syncs data between tables
* ‚úÖ Reduces app-side logic

---

## ‚ö†Ô∏è Cautions with Triggers

* ‚ùó Can impact performance if misused
* ‚ùó Not suitable for complex logic
* ‚ùó Harder to debug compared to application code
* ‚ùó Avoid recursive or cascading triggers unless absolutely necessary
